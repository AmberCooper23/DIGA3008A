Amber Alert

Reflection on the Implementation of the Website

Throughout the semester, I worked on a website called Amber Alert. The goal of the website was to showcase my development journey, as well as my work. My website features weekly blogs where I upload my weekly discussions, a design section where I discuss the design processes and elements that went into the website, an essay section for the essays I wrote this semester for this class, and a portfolio section to showcase things I am proud of.

Folder Structure

An underestimated part of web development is folder structure, as this can make or break your work. I learnt that you need to structure your folders in a way that makes sense to you, but would also make to someone else, making the project folder understandable to others. It also matters that you don’t have too many folders, and that your files are placed systematically, so that you don’t have issues with referencing.

My folder structure separates the different html pages into the sections of my website, and I kept my CSS and JS in their own respective folders. I also kept all my Images in their own folder and then made a separate folder inside the Images folder for the art that goes in my portfolio. This allowed for my work to stay organised and sensical to me. I also made sure to be consistent in the naming conventions, keeping everything in camel case.

Each section homepage html file is named as [sectionName]Page to separate it from everything else.

When it comes to my weekly blogs, I have chosen to just title the html files by the week. The whole point of the website is to showcase what I learn every week, so it made a lot of sense to keep it as the week numbers rather than giving each week its own titles. This works out better in the long run when there are many more weekly blogs, as they can all be found under blogs. If it gets to that point, I might have to consider subfolders, as that adds another level of organisation.

My CSS file names are consistent with the whole project, naming homepages as [sectionName]Page. I believe that consistency in labels and names is incredibly important and avoids a lot of confusion and mixing up. Having clear labels in a project is so helpful when working in teams, to avoid multiple working on the same file simultaneously.

Meta Data

Meta Data communicates the important information for search engines. This ensures your site functions, can be found, and is linked to your name as the author.

The meta tag is used for metadata elements that don’t have their own tags (title, link, script, etc). I used for charset, viewport, author, description.
